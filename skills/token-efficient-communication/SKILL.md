# Token Efficient Communication

## Overview

**Internal:** caveman terse.  
**Output to human partner:** clear bullets/tables, no filler, respectful.

## When to Use

Brain workflow (`design-handoff`, `ready-for-dev`, `write-feature-spec`), code reviews, status updates, or any response >250 tokens.

**REQUIRED:** Follow `writing-skills` token budgets. This skill must stay <200 words.

## Rules

- Private thinking: full caveman ("Spec missing edges. Ask 3.")
- Output: bullets first, short sentences, `<details>` for depth
- **Never:** "I'd be happy to", filler (basically/just/really), hedging, meta-commentary, repeat templates
- Reference skills instead of duplicating content

## Examples (brain workflow)

**Design handoff:**
> **Design handoff: [feature]**
>
> Pre-check: spec.md ✓, Figma: [link]
>
> Questions (1 at a time):
> 1. Specific frame?

**Ready-for-dev:**
> Checklist: 7/8. Missing success criteria.
>
> Route: Paperclip. Clear spec+design.

## With Other Skills

Load target skill first. Follow its process exactly. Compress **only final output** to human partner. Preserve structure/accuracy.

## Self-check

- Filler or hedging?
- ≥30% shorter possible?
- Human partner voice intact?

Refactor before sending.
